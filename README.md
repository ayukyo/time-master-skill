# 刘亚格.skill - 海王模式 - 时间管理大师

> 多线并行自动化 Skill for OpenClaw

---

## 起源：一个真实案例

本 Skill 的灵感来源于一份真实的社会事件材料 —— **"洛阳第一女海王刘亚格"**事件。

> 刘亚格，武汉某大学法学院毕业，洛阳体制内工作，同时与三名男性保持恋爱/订婚关系长达两年，期间实现了：
> - 三线时间线完全并行，无重叠冲突
> - 每个男友都不知道其他人的存在（**完美上下文隔离**）
> - 工作日、周末、假期精确分配到人（**时间分片调度**）
> - 每人一套专属话术体系（**模板×变量输出**）
> - 查岗时随时切换状态（**优先级动态调度**）
> - 假期多天安排三人无撞车（**冲突检测**）
> - 甚至其母亲全程知情并配合（**监督者模式**）

抛开道德评判，这套**多线并行、上下文隔离、零冲突切换**的操作模式，恰好是 AI 自动化中**多任务并行调度**的经典需求。

## 它能做什么

| 场景 | 刘亚格原版操作 | OpenClaw 自动化 |
|------|--------------|-----------------|
| **多账号运营** | 两个微信号隔离三组关系 | 同时登录多个微信/钉钉/邮箱，各账号上下文完全隔离，切账号=切人格 |
| **多项目管理** | 工作日陪一个男友，周末陪另外两个 | 周一至三处理 Project-A，周四至五处理 Project-B，周末批量跑维护任务 |
| **就近资源复用** | 三个对象在同一城市活动，减少切换成本 | 同类任务放同一目录/同一窗口，减少路径切换开销 |
| **差异化输出** | 对每个对象用不同话术体系 | 同一份日报，自动适配周报/日报/摘要格式，语气按接收方调整 |
| **紧急插入** | 哪个对象查岗紧就先安抚谁 | 高优任务（如线上告警）插入时，快照当前任务 -> 处理告警 -> 恢复原任务 |
| **排期防冲突** | 假期分成三段，分别安排给三个人 | 多个定时任务自动检测时段重叠，建议错峰执行 |
| **优雅降级** | 回不了消息就"心情不好/在开会" | API 超时时自动切换到备选方案，记录阻塞原因 |
| **操作审计** | 事后梳理时间线发现真相 | 调度日志自动记录所有操作轨迹，精确到秒级时间线 |
| **监督者模式** | 母亲全程知情 | 主 Agent 监控子 Agent 状态，异常时自动告警 |
| **通道隔离** | 大小号分别联系不同对象 | 敏感操作走专用通道，日常操作走公共通道 |

## 核心概念

### 上下文隔离（双微信号模式）

```
[会话-A: 男友A]
  微信大号 | 工作日模式 | 同居住所
  -> 聊结婚买房、见家长

[会话-B: 男友B]
  微信小号 | 周末模式 | 老家/同居处
  -> 聊日常、共同生活

[会话-C: 男友C]
  微信大号 | 周末+假期 | 另一处住所
  -> 聊订婚、彩礼、看房
```

在 OpenClaw 中：每个上下文独立存储状态、凭证、会话信息。切换到上下文-B 时，绝对不会误操作到上下文-A 的资源。

### 时间分片（工作日/周末模式）

```
周一至五（工作日）：
  └── 男友A：同居、上下班、见家长

周六至日（周末）：
  ├── 男友B：回老家、同居、旅游
  └── 男友C：另一住处约会

假期封神安排：
  第1-3天 ── 男友B（旅游）
  第4-6天 ── 男友C（见家长+旅游）
  第7-9天 ── 男友A（工作日+短途出游）
```

在 OpenClaw 中：任务按"时间片"分配。每个时间片有明确的任务目标和上下文，时间片之间强制快照+恢复，不留痕迹。

### 优先级调度（查岗安抚）

```
场景：男友A晚上突然视频查岗
  1. 挂断男友B的视频（快照当前上下文）
  2. 切换到"男友A"模式（调整语气/环境）
  3. 说"心情不好，刚在洗澡"（阻塞借口）
  4. 安抚完毕（任务完成）
  5. 切回男友B（恢复上下文）
```

在 OpenClaw 中：高优任务到达时，自动保存当前任务状态->切换到高优任务->完成后无缝恢复。

### 冲突检测（十一假期模式）

```
输入：
  男友-A：第1-3天 旅游
  男友-B：第4-6天 旅游
  男友-C：第7-9天 旅游+出游

输出：
  [OK] 完美！时间线无重叠
  [WARN] 注意：第7天是工作日但也和C在一起
      建议提前申请年假
```

在 OpenClaw 中：多个定时任务提交时，自动检查时段重叠。发现冲突时给出错峰建议。

## 用法示例

### 示例 1：多账号内容发布

```
用户：帮我把这份通知发到三个渠道，用 time-master 模式

刘亚格模式启动：
[上下文-A: 企业微信] -> 格式化成正式通知，@全员
[上下文-B: 钉钉群]   -> 格式化成群公告，带进度表
[上下文-C: 邮件]     -> 格式化成邮件正文，含附件
────────────────────────────────
时间分片：
  |- 09:00 企业微信（先发，最重要的渠道）
  |- 09:15 钉钉群（复制+调整格式）
  └─ 09:30 邮件（需打包附件，单独处理）
────────────────────────────────
三线汇总：
  [OK] 企微 -> 已发送，3 人已读
  [OK] 钉钉 -> 已发送，群公告置顶
  [OK] 邮件 -> 已发送，收件人 12 人
```

### 示例 2：多项目周报生成

```
用户：同时生成 A/B/C 三个项目的周报

刘亚格模式启动：
[上下文-A: 项目Alpha] -> 迭代v2.1 进度 80%，风险：无
[上下文-B: 项目Beta]  -> 性能优化 60%，阻塞：等待QA环境
[上下文-C: 项目Gamma] -> 重构 30%，风险：人员不足
────────────────────────────────
模板统一，变量不同：
  "本周 {project} {progress}，风险：{risks}，下周计划：{plan}"
────────────────────────────────
三线汇总：
  [OK] Alpha -> 80%，下周提测
  [WAIT] Beta  -> 阻塞中（Plan-B：先做单元测试）
  [OK] Gamma -> 30%，已申请增援
```

### 示例 3：定时任务排期冲突检测

```
用户：帮我排一下这周的定时任务
  任务1：每天 9 点备份数据库（约 20 分钟）
  任务2：每天 9 点生成日报（约 10 分钟）
  任务3：周二 9:30 周会提醒

刘亚格模式 -> 冲突检测：
  [WARN] 每天 9:00-9:20 备份与日报重叠！
    建议：备份提前到 8:30，日报改到 9:30
  [WARN] 周二 9:30 周会与日报冲突！
    建议：周二日报改到 10:00
```
