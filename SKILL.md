# 刘亚格·时间管理大师 — Time Master Skill

## 适用场景

当你需要 OpenClaw 同时处理**多个独立上下文**的任务时：

| 场景模式 | 自动化映射 |
|---------|-----------|
| 多账号隔离操作 | 同时操作多组账号/客户端，上下文完全隔离 |
| 时间分片执行 | 不同时段执行不同类型任务 |
| 紧急任务插入 | 高优任务打断时，快照→处理→恢复 |
| 排期冲突检测 | 多个定时任务自动检测时段重叠 |
| 模板变量输出 | 同一数据适配多格式/多接收方 |
| 优雅降级 | 任务阻塞时自动切换备选方案 |

## 行为指令

### 1. 上下文隔离

为每个独立任务线创建显式命名的上下文：

```
Context-A: [任务名称] — [凭证/工作目录/状态]
Context-B: [任务名称] — [凭证/工作目录/状态]
```

- 每个上下文独立管理状态和凭证
- 切换上下文时，先输出当前上下文摘要再切换
- **严禁上下文串扰**：A 的数据不得出现在 B 的处理中

### 2. 时间分片

为每个上下文分配时间片：

```
时间片轮转：
  ┌── 时段1 ── Context-A（深度工作）
  ├── 时段2 ── 缓冲区（应对突发插入）
  ├── 时段3 ── Context-B（批量任务）
  └── 时段4 ── 汇总报告
```

- 时间片之间输出分割线 `---`
- 高优任务可抢占当前时间片，完成后回到原上下文

### 3. 优先级动态调度

```
当紧急度 > 当前任务优先级：
  1. 快照当前上下文（输出当前进度）
  2. 切换到高优上下文
  3. 完成后恢复原上下文
```

- 紧急度评估因素：deadline 紧迫度、调用方权重
- 所有切换记录到调度轨迹

### 4. 冲突检测

当收到多个定时任务或时间敏感任务时，先做冲突检测：

```
输入：Task-A: 10/1-10/3, Task-B: 10/2-10/4
输出：时段重叠告警 + 错峰建议
```

检测脚本：`scripts/conflict-detector.py`（支持 stdin yaml/json 输入）

### 5. 模板×变量输出

```
模板: "本月 {project} 进度 {percent}%，{next_action}"
分别代入各上下文变量，输出差异化版本
```

### 6. 优雅降级

```
Plan-A（主路径）→ 重试 3 次
Plan-B（备选）  → 执行降级操作
Plan-C（记录）  → 记录阻塞原因，通知调用方
永不硬等 → 超时即切换
```

## 输出格式

每次多线操作结束后，必须输出**汇总报告**：

```
=== 多线汇总 ===
[Context-A] 状态（耗时/切换次数）
[Context-B] 状态（阻塞原因/降级措施）
[Context-C] 状态

调度轨迹：
  时段1 ── Context-A: 操作简述
  时段2 ── Context-C: 突发插入（原因）
  时段3 ── Context-A: 恢复执行
```

## 激活方式

以下 prompt 意图自动触发：
- "同时操作多个账号/窗口"
- "多线并行" / "时间分片"
- "多个项目隔离"
- "防冲突调度"
- 显式调用：`/time-master`

## 配合的其他技能

| 技能 | 配合方式 |
|------|---------|
| browser | 每标签页一个独立上下文 |
| exec | 各项目工作目录隔离执行 |
| web_search | 多关键词并行搜索，按上下文归类 |

## 限制

- 并行上下文不超过 5 个
- 显式命名每个上下文，避免 Agent 混淆
- 频繁上下文切换不如排队串行
- 调度日志保留最近 100 条
